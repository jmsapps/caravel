# C-6 Production Hardening

Architecture decision, 2026-07-04: bare Caravel is a small deterministic
execution core with no mandatory control-plane metadata; checkpoints, run
evidence, ownership pruning, and leases become explicit plugins (see
`.skills/tasks/C-6/SKILL.md`). The earlier Phase 1 checkpoint implementation
(branch `C-6-phase-1`, commit `f18574d`) was rolled back and is being re-landed
as core data-safety behavior now plus a first-party CheckpointPlugin later. Its
previously announced breaking changes apply only as re-introduced below.

## Phase 1, Step 1.1 — Core Data-Safety Port

- Breaking: `StepContext.run_id` is now a per-run UUID4 hex string instead of
  the run-root basename.
- Built-in datasets validate the complete payload (shape, partition keys,
  record types) before touching any output; a validation failure preserves
  prior output.
- The runner replaces the Caravel-owned step directory before every persisted
  built-in save, so fewer-key reruns leave no stale partition files. Custom
  datasets without `validate_payload` keep plain save behavior.
- Breaking: `clean_dirs=True` without `stage_root` now cleans only the resolved
  stage directory instead of the entire run root; foreign files in the run root
  and other stages' outputs survive.
- The `.caravel_empty` marker is unchanged in this step; its removal lands with
  the executor rewrite (Step 1.3), together with the legacy file-existence
  selective path that consumes it.

## Phase 1, Step 1.2 — Logical Plan Compiler

- Added `caravel.plan` with `compile_pipeline(pipeline) -> LogicalPlan`:
  immutable nodes over the closed set {step, fan-out, route-step, merge},
  index-based rename-stable node IDs, explicit input references, and
  byte-stable JSON snapshots. Compilation knows no run root, selector,
  storage, or plugin state.
- Compilation validates declaration facts: path-safe pipeline/stage/step/
  branch/route identifiers, decorated route callables, supported branch
  shapes, non-empty route maps, and duplicate route-step names within one
  route (whose output paths would collide on disk today). These checks take
  execution effect when the runner adopts the plan in Step 1.3.
- The runner is unchanged in this step.

## Phase 1, Step 1.3 — Runtime Binding And Single Executor

- The runner now compiles the declaration (`compile_pipeline`), binds it to an
  `ExecutionRequest` (`bind_execution`), and executes every node kind through
  one executor dispatch. Binding is the only layer that knows run roots and
  selectors; the public `run(...)` signature is unchanged.
- Breaking: selective execution fails closed. A selective request that must
  load output from a node it does not execute raises
  `UnsupportedCapabilityError` (a `MissingPriorOutputError` subclass) at plan
  binding, before any user code or mutation. Bare core never trusts existing
  output files as evidence of completed work; checkpoint-backed selective
  execution returns with the CheckpointPlugin (Phase 2). Selective requests
  that need no prior output (first stage, or step 1 of the first stage) still
  execute from the loader seed.
- Breaking: full runs pass outputs in memory, including across stage
  boundaries. Stage terminals no longer need to be persisted, and payloads no
  longer round-trip through JSON between stages. A non-dict payload still
  cannot cross a stage boundary.
- Breaking: `.caravel_empty` is removed everywhere; Dataset output
  directories are data-only. On backends without durable empty directories a
  committed-empty partitioned output leaves no loadable artifact; durable
  empty-output evidence is a checkpoint-plugin concern. Partitioned
  `exists()` is now false for empty output.
- Fixed: a step declared after a Branch in the same stage now receives the
  merged branch output; previously it received the payload of the step before
  the branch.
- Compiler validation from Step 1.2 (path-safe names, decorated route
  callables, non-empty route maps, duplicate route-step names within a route)
  now applies to every run.

## Phase 0 — Reproducible Safety Net

- Added clean-checkout CI for Python 3.11 through 3.14.
- Added a Python 3.11 job that tests the minimum supported fsspec version.
- Added explicit ruff formatting/linting and strict mypy gates.
- Added test and example dependency extras; python-dotenv is no longer a core
  runtime dependency.
- Added the packaged py.typed marker and exported package metadata version.
- Documented AGENTS.md and .skills as intentionally local-only.
