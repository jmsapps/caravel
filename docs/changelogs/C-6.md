# C-6 Production Hardening

## Phase 0 — Reproducible Safety Net

- Added clean-checkout CI for Python 3.11 through 3.14.
- Added a Python 3.11 job that tests the minimum supported fsspec version.
- Added explicit ruff formatting/linting and strict mypy gates.
- Added test and example dependency extras; python-dotenv is no longer a core
  runtime dependency.
- Added the packaged py.typed marker and exported package metadata version.
- Documented AGENTS.md and .skills as intentionally local-only.

## Phase 1 — Central Checkpoint Registry And Replace Safety

- Added a central checkpoint registry: one schema-versioned record per
  persisted node under
  `<run_root>/<pipeline_name>/_000_metadata/checkpoints/<node-id>.json`. Step
  output directories now contain dataset files only.
- Each run generates a UUID4 run ID. `StepContext.run_id` now carries it
  instead of the run-root basename (deliberate breaking change).
- Runner-persisted saves follow validate → invalidate record → replace output
  → publish record last. An interrupted replacement is detectably uncommitted;
  selective reruns refuse it with `MissingPriorOutputError`.
- Selective boundaries verify record/declaration/physical-output agreement.
  New public errors: `CheckpointError`, `CheckpointIntegrityError`,
  `UnsupportedCheckpointVersionError`, `CheckpointCommitUncertainError`.
- Removed the `.caravel_empty` sentinel (deliberate breaking change, no
  migration). An allowed empty partitioned output is a count-zero record.
- `clean_dirs` without `stage_root` now cleans only the stage's own directory
  — never the run root or other stages (deliberate breaking change).
- Bare `dataset.save()` output without a record is ordinary data, not a
  reusable checkpoint boundary.
- Added `Pipeline.metadata_storage_options` for the control store; storage
  options are never serialized into records.
- Deterministic failpoints cover every commit boundary; the contract suite
  (`tests/caravel/test_checkpoints.py`) runs failure injection, tampering,
  fewer-key replacement, empty checkpoints, and `stage_root` cases on local
  paths and `memory://`.
